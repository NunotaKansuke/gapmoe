# Notes Policy

This repository is being prepared for public release. Keep `.note/` updated as work proceeds.

Rules:

- Record the current state after each meaningful structural or design change.
- Write down decisions, unresolved issues, and next steps before moving on.
- Keep notes factual and dated when useful.
- Prefer updating an existing note when it is the right home; add a new note only when the topic deserves its own file.
- Do not treat `.note/` as public user documentation. It is the working log for repository cleanup and design decisions.
